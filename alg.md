- 当前的逻辑本质

  > 训练计划 = 时间维度递增 + 负荷递增 + 动作复杂度递增
  >
  > 
  >
  > 核心假设：
  >
  > 1. 人体会线性适应训练刺激
  > 2. 只要持续递增刺激，能力就会提升
  > 3. 目标由“平均刺激”驱动（默认健康目标）
  > 4. 训练计划是一个单向进化过程

- 人体不是刺激驱动系统。而是：

  > 刺激 → 疲劳 → 恢复 → 超量恢复 → 适应 → 新基线

- 当前模型存在的问题。
  1. 是“刺激递增模型”，不是“适应管理模型”。

     当前算法只建模了`刺激`这第一个环节。而缺失：

     - 疲劳积累函数

     - 恢复窗口函数

     - 适应滞后

     - 过载阈值

     因此，会产生：“过载失控”或“刺激不足”的两极波动。

  2. 没有建立“容量上限”概念。

     任何个体都存在：

     1. 最大恢复容量（MRV）
     2. 最小有效刺激（MEV）

     真实世界个体：

     - 刺激 < MEV  → 无效
     - 刺激 > MRV → 过载

  3. 没有定义“训练密度”。

     训练密度 = 单位时间内完成的有效训练量

     例如：

     - 30分钟做20组
     - 30分钟做10组

     > Volume（量） × Intensity（强度） ÷ Time（时间） = Density（密度）

     密度是控制：

     - 代谢压力
     - 心肺负荷
     - 恢复需求

     如果不建模密度，永远无法控制疲劳曲线。

  4. 没有“波动模型”。

     人体适应是波动曲线，

     > 上升 → 平台 → 微下降 → 再上升

     所以训练系统必须存在：

     - 周期卸载（Deload）
     - 强弱交替
     - 微周期调节

  5. “刺激模型”是算法结构选择器。是算法模板。

     - 肌肥大模型
     - 力量模型
     - 代谢模型
     - 神经控制模型
