# 训练计划算法-原则

------

# Layer 0：基石原则

这些原则不随用户目标改变，不随周期改变，不随阶段改变。

它们决定系统的价值函数。

------

## P1. 安全优先于效率

任何训练计划生成必须满足：

- 负荷控制在恢复能力范围内
- 避免长期疲劳累积
- 避免高风险技术动作的滥用

若安全与效率冲突，优先降低效率。

------

## P2. 长期可持续优先于短期突破

系统目标不是“最快进步”，而是：

> 在最小伤害风险下实现长期稳定进化。

------

## P3. 可解释性原则

任何输出的训练计划必须：

- 能够说明刺激逻辑
- 能够说明负荷递增逻辑
- 能够说明周期设计逻辑

黑箱算法不符合本系统原则。

------

## P4. 结构优于经验

算法结构优先于个人经验偏好。

- 不依赖教练个人习惯
- 不依赖动作偏爱
- 依赖变量计算与权重分配

------

## P5. 简洁优于复杂

在效果接近情况下：

- 选择变量更少的模型
- 选择更易执行的动作
- 选择更稳定的周期结构

避免“为了专业而复杂”。

------

# Layer 1：算法设计原则

这些是方法论级原则。

------

## D1. 渐进过载原则

训练刺激必须在时间维度上递增或变化。

但递增必须在安全区间内。

------

## D2. 恢复约束原则

刺激必须小于个体恢复上限（MRV）。

任何连续负荷递增必须伴随波动与卸载。

------

## D3. 适应特异性原则（SAID）

训练刺激必须与目标方向一致。

- 增肌 → 容量与张力优先
- 力量 → 强度与神经优先
- 减脂 → 密度与代谢优先

------

## D4. 动作质量优先原则

动作控制优先于负荷增加。

当质量下降：

- 优先调整动作
- 再调整负荷

------

## D5. 变量统一计量原则

所有训练负荷必须能够被量化为统一变量体系：

- Volume
- Intensity
- Density
- Frequency

禁止不可量化刺激。

------

## D6. 波动优于线性原则

训练负荷必须呈现周期波动。

禁止无限线性递增模型。

------

## D7. 默认均衡原则

在无明确短板目标时：

> 采用全身均衡刺激模型。

但均衡是权重分配，而非动作平均。

------

# Layer 2：算法行为风格原则

这是算法产品的“人格”。

------

## A1. 默认保守进化

在缺乏充分反馈数据时：

- 选择中等负荷
- 选择可恢复区间
- 选择低风险动作

------

## A2. 客观数据优先于主观判断

若存在：

- RPE
- 完成度
- 心率
- 疲劳评分

则以数据修正模型，而非固定模板。

------

## A3. 自适应优先于固定模板

固定模板仅在：

- 数据缺失
- 新手阶段

作为过渡方案。

------

# Layer 3：算法实现约束原则

这些不是训练原则，而是算法边界。

------

## I1. 单节课负荷必须可计算

系统必须能够输出：

```
SessionLoad = f(Volume, Intensity, Density)
```

------

## I2. 周负荷必须可计算

```
WeeklyLoad = Σ(SessionLoad)
```

------

## I3. 周期波动必须可计算

```
Load(t) = Base × 波动函数
```

------

## I4. 禁止无限递增

所有递增必须：

- 有上限
- 有卸载
- 有恢复窗口
