
# 自适应双轨健身课程编排架构

**Adaptive Dual-Track Fitness Programming Architecture**

## 1. 顶层设计原则 (The Axioms)

本系统架构建立在运动科学三大基石之上，作为不可违背的“物理法则”：

* **FITT-VP 原则 (ACSM)**：严格控制频率(F)、强度(I)、时间(T)、类型(T)、容量(V)与进阶(P)。
* **OPT 模型 (NASM)**：遵循 **稳定性 (Stabilization)  力量 (Strength)  爆发力 (Power)** 的生理适应顺序。
* **SAID 原则 (特异性适应)**：训练目标决定训练手段，“练什么得什么”。

---

## 2. 核心路由算法 (Routing Algorithm)

**逻辑判定：** 优先响应用户的**生理目标 (Intent)**，根据**环境约束 (Constraints)** 动态计算最佳执行路径。

### 2.1 决策流程

1. **Input:** 用户主目标（增肌/力量 OR 减脂/体能）
2. **Check:** 可用资源（健身房/固定器械 OR 居家/小器械）
3. **Process:** 调用 决策矩阵 (Decision Matrix)
4. **Output:** 执行协议 (A 或 B)

### 2.2 决策矩阵 (Decision Matrix)
| **用户目标 (Intent)**            | **环境：健身房 (Gym)**                             | **环境：居家/小器械 (Home)**                       |
| -------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **增肌 / 力量** *(追求机械张力)* | **【原生模式 - Native】** 调用 **协议 A (标准版)** *逻辑：大重量孤立刺激* | **【仿真模式 - Emulation】** 调用 **协议 B (增肌变种)** *逻辑：增加TUT、单边负荷、慢速离心* |
| **减脂 / 体能** *(追求代谢压力)* | **【仿真模式 - Emulation】** 调用 **协议 A (减脂变种)** *逻辑：小重量多动作、超级组、0间歇* | **【原生模式 - Native】** 调用 **协议 B (标准版)** *逻辑：全身功能性循环、高心率* |
---

## 3. 执行协议层 (Execution Protocols)

### 3.1 协议 A：结构化抗阻 (Structural Resistance)

*适用场景：健身房 / 固定器械 / 哑铃区*

* **解剖逻辑**：**肌群分化 (Split Routine)**
* 典型结构：三分化（推/拉/腿）、五分化（胸/背/肩/臂/腿）。


* **量化标准**：**次数 (Repetition-based)**
* 范围：8-12 RM（增肌）、15-20 RM（耐力/减脂）。


* **强度锚定**：**力竭 (Mechanical Failure)**
* 利用器械安全性，冲击肌肉生理极限。


* **动作筛选**：以孤立肌群收缩为主（如：坐姿推胸、腿屈伸）。

### 3.2 协议 B：功能性体能 (Functional Performance)

*适用场景：居家 / 徒手 / 小器械 / 弹力带*

* **解剖逻辑**：**动作模式 (Movement Patterns)**
* 六大模式矩阵：`Push` (推), `Pull` (拉), `Squat` (蹲), `Hinge` (髋铰链), `Lunge` (剪蹲), `Rotate` (旋转/抗旋)。
* 典型结构：全身整合 (Full Body) 或 上下肢分化 (Upper/Lower)。


* **量化标准**：**时间 (Time-based / TUT)**
* 范围：30s - 45s / Set。
* *逻辑：在负重受限时，通过拉长受力时间募集慢肌纤维。*


* **强度锚定**：**RPE (Rate of Perceived Exertion)**
* 阈值：RPE 7-9。
* *逻辑：利用循环训练（Circuit Training）和短间歇（0s-30s）累积代谢压力。*

## 4. 课程微观结构 (Session Architecture)

单节课程遵循“三段式”标准模型，如同一个完整的 I/O 流程：

### Phase 1: 输入预热 (Input / Prep)

* **SMR (筋膜放松)**：抑制过活跃肌肉（泡沫轴）。
* **Dynamic Stretch (动态拉伸)**：恢复肌肉长度。
* **Activation (神经激活)**：预先唤醒核心及稳定肌群（如：死虫式、弹力带行走）。

### Phase 2: 核心处理 (Process / Main)

* **Block 1 (主项)**：根据协议 A 或 B 执行核心训练。
* **Block 2 (辅项/补强)**：针对薄弱环节或代谢收尾。

### Phase 3: 输出冷却 (Output / Recovery)

* **Static Stretch (静态拉伸)**：副交感神经激活，降低肌张力。
* **Cool-down (整理)**：心率回落。

## 5. 进阶路线图 (Progression Roadmap)

用户全生命周期的演变逻辑，遵循依赖解锁机制：

* **Level 1: 基础期 (Stabilization & Alignment)**
* **关键词**：体态、核心、慢速。
* **逻辑**：修复Bug（代偿），建立系统稳定性。
* *典型课表：居家小器械-体态稳定系列。*


* **Level 2: 进阶期 (Endurance & Mobility)**
* **关键词**：代谢、心肺、大幅度。
* **逻辑**：提升系统吞吐量（心肺功能）和带宽（关节活动度）。
* *典型课表：居家徒手-疲劳改善/燃脂系列。*


* **Level 3: 强化期 (Strength & Coordination)**
* **关键词**：负荷、爆发、复合动作。
* **逻辑**：系统压测，追求峰值性能。
* *典型课表：固定器械-力量循环系列。*



**线性周期参数 (Linear Progression Example):**

> 在协议 B 中，动作容量按 `30s`  `35s`  `40s`  `45s` 的阶梯逐周递增。

---

## 6. 反馈与闭环 (Feedback Loop)

由于缺乏实时生物数据监控，系统采用“主观传感器”进行闭环控制。

* **Sensor (传感器)**：用户反馈的 **RPE (1-10)**。
* **Controller (控制器逻辑)**：
* `IF` RPE < 目标区间 (e.g., <7)：
* Gym: 增加 5% 重量。
* Home: 增加 5s 动作时间 或 缩短 10s 组间歇。


* `IF` RPE > 目标区间 (e.g., >9 且动作变形)：
* Gym: 降低重量。
* Home: 退阶动作难度 (e.g., 俯卧撑  跪姿俯卧撑)。
