# 训练计划算法-逻辑

## 逻辑

> 训练计划 = f(用户目标, 当前能力, 约束条件, 时间维度)

即，

> 目标驱动 → 模型选择 → 边界控制 → 波动设计 → 课程生成


- 人体不是刺激驱动系统。而是：

  > 刺激 → 疲劳 → 恢复 → 超量恢复 → 适应 → 新基线

因此，训练计划算法必须是“适应管理模型”。

  1. 要是“适应管理模型”

     不能只建模`刺激`这一个环节。也要包括：
     - 疲劳积累函数
     - 恢复窗口函数
     - 适应滞后
     - 过载阈值

     否则，会产生：“过载失控”或“刺激不足”的两极波动。

  2. 要建立“容量上限”概念

     任何个体都存在：
     1. 最大恢复容量（MRV）
     2. 最小有效刺激（MEV）

     真实世界个体：
     - 刺激 < MEV  → 无效
     - 刺激 > MRV → 过载

  3. 要定义“训练密度”

     训练密度 = 单位时间内完成的有效训练量

     例如：
     - 30分钟做20组
     - 30分钟做10组

     > Volume（量） × Intensity（强度） ÷ Time（时间） = Density（密度）

     密度是控制：

     - 代谢压力
     - 心肺负荷
     - 恢复需求

     如果不建模密度，永远无法控制疲劳曲线。

  4. 要有“波动模型”

     人体适应是波动曲线，

     > 上升 → 平台 → 微下降 → 再上升

     所以训练系统必须存在：

     - 周期卸载（Deload）
     - 强弱交替
     - 微周期调节

  5. “刺激模型”是算法结构选择器。是算法模板。

     例如：
     - 肌肥大模型
     - 力量模型
     - 代谢模型
     - 神经控制模型
